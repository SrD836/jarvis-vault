# Code Audit — 2026-05-29

> Auditoria PARALELA autonoma (`hermes/overnight_audit.py`, DeepSeek). FASE 5. Clusters auditados: observability, llm-routing, memory, vault-maintenance, autonomous-loops. Fallidos: ninguno.

## Resumen ejecutivo

Se auditaron 5 clusters (observability, llm-routing, memory, vault-maintenance, autonomous-loops) identificando 27 hallazgos: 9 de severidad ALTA, 10 MEDIA y 8 BAJA. No existen auditorías previas, por lo que no hay hallazgos recurrentes. Los riesgos más críticos incluyen pérdida de datos por operaciones no atómicas, bloqueos indefinidos por falta de timeouts, inyección FTS5, y errores de sintaxis que impiden la ejecución de módulos completos.

## Hallazgos por severidad

**ALTA**

## Hallazgos por cluster (crudo)

### observability
`run_logger.py` · ALTA · `_maybe_rotate` no es atómica entre `os.path.getsize` y `os.replace`; si dos procesos rotan simultáneamente, se pierden datos. Usa lock de archivo o rename con PID.  
`run_logger.py` · MEDIA · `append_line` usa `os.write` sin reintentar en `EINTR`; en cron con señales puede truncar la línea.  
`run_logger.py` · BAJA · `_run_wrapper` captura `Exception` genérico, pero `subprocess.run` con `capture_output=True` puede lanzar `FileNotFoundError` específico; no distingue fallos del comando vs. del wrapper.  
`runs_dashboard.py` · ALTA · `parse_vault_runs` itera `agents_dir.iterdir()` sin filtrar entradas no-directorio; si hay un archivo suelto, falla con `AttributeError` en `is_dir()`.  
`runs_dashboard.py` · MEDIA · `field()` recompila regex en cada llamada; en directorios con muchos `.md` es O(n*m) innecesario.  
`runs_dashboard.py` · BAJA · `JOBS_PATH` se lee completo en cada ejecución; si el archivo es grande (>1MB), es derroche de I/O. Cachear o leer solo si cambió.

### llm-routing
`llm_call.py` · ALTA · `_call_ollama` y `_call_claude` no tienen timeout en `subprocess.run` (solo en firma, no se pasa). Si el contenedor cuelga, el cron se bloquea indefinidamente.

`llm_call.py` · ALTA · `_try_telegram` parsea token de `.env` pero no valida que exista; si falta, falla silenciosamente y no se notifica fallo de modelo.

`llm_call.py` · MEDIA · `_call_deepseek` usa `urllib.request.urlopen` sin `context=ssl.create_default_context()`. En VPS con proxies corporativos o certificados custom puede fallar.

`llm_call.py` · MEDIA · `_call_ollama` y `_call_claude` no sanitizan `system`/`user` antes de pasarlos como input a `docker exec`. Inyección de comandos shell posible si el contenido contiene caracteres de control.

`llm_call.py` · BAJA · `load_env` no maneja archivos con BOM o líneas con espacios alrededor del `=`. Puede ignorar variables válidas.

`llm_call.py` · BAJA · `NOTIFY_STATE` y `FAIL_LOG` se escriben en `LOGS_DIR` sin verificar que exista. Si se borra, falla con `FileNotFoundError`.

### memory
**memory_sqlite.py**
- `memory_sqlite.py` · ALTA · `_fts_query()` no escapa comillas dobles dentro de tokens · Permite inyección FTS5: un token como `"OR ""` puede romper sintaxis o exponer datos no autorizados.
- `memory_sqlite.py` · MEDIA · `dedupe` usa `LIKE` con hash en metadata JSON · Falso positivo si otro campo contiene el hash; debería usar `json_extract(metadata, '$.hash')`.
- `memory_sqlite.py` · BAJA · `store()` acepta `agente` y `tipo` desde `meta.pop()` · Puede sobrescribir argumentos explícitos si metadata los contiene, comportamiento sorpresivo.

**memory_backfill.py**
- `memory_backfill.py` · MEDIA · `row_contenido()` usa `f"col{i}"` como key si header más corto que cells · Produce metadata con keys genéricas que contaminan la DB; debería truncar o saltar.
- `memory_backfill.py` · BAJA · `PROFILE_AGENTS` hardcodeado · Si se añade agente profile, hay que modificar código; debería ser configurable o detectarse por convención.

**memory_export.py**
- `memory_export.py` · MEDIA · `connect_ro()` con `mode=ro` no maneja DB inexistente · `main()` chequea existencia pero si se borra entre check y connect, crashea; debería capturar excepción.
- `memory_export.py` · BAJA · `safe = agente.replace("/", "_")` · Path traversal mitigado pero no validado; agente como `../etc` crearía archivo fuera de OUT.

**memory_inject.py**
- `memory_inject.py` · ALTA · `except Exception: pass` en import de `memory_sqlite` · Silencia errores reales (SyntaxError, ImportError) que deberían detener ejecución; usar `ImportError` específico.
- `memory_inject.py` · MEDIA · `except Exception: summary = ""` en `summarize_for_prompt` · Silencia todos los errores; un fallo DB se oculta y se usa fallback legacy sin log.
- `memory_inject.py` · BAJA · `ALL_AGENTS` hardcodeado · Desincronizado con agentes reales en filesystem; debería descubrirse dinámicamente.

## Hallazgos

**linker.py** · ALTA · `deepseek_suggest()` usa `json.loads()` sin `strict=False` ni manejo de `json.JSONDecodeError` · Si DeepSeek devuelve texto extra (markdown, trailing comma), el script crashea sin fallback, dejando archivos sin enlazar.

**linker.py** · MEDIA · `build_vault_index()` ordena por mtime pero no persiste el índice · Cada ejecución re-escanea todo el vault (250 archivos) y re-llama a DeepSeek con el mismo índice, desperdiciando tokens y tiempo.

**curator.py** · ALTA · `check_probation()` truncado en medio de condicional (`if failure_streak == 0 and current`) · SyntaxError garantizado en ejecución, el módulo no corre.

**curator.py** · MEDIA · `emit_skill_log()` usa `now_iso()` no definida en el código · NameError en tiempo de ejecución cuando se intenta loguear.

**orgchart.py** · BAJA · `role_from_subagents()` lógica confusa: `if allow: return "orchestrator"` pero luego `if allow == []: return "worker"` · `[]` es falsy, el segundo branch nunca se ejecuta; todos con `allow=[]` se marcan como orchestrator.

**orgchart.py** · MEDIA · `load_registry()` itera `runs.values()` pero si `runs` es lista (posible según comentario) falla con AttributeError · Sin try/except en ese nivel, el script entero muere.

**orgchart.py** · BAJA · `preserve_human_notes()` regex `re.DOTALL` captura desde `## Human notes` hasta el final del archivo · Si hay otro `##` después, se lo traga todo.

### autonomous-loops
`nightly_digest.py` · ALTA · `recall_memories` llama a `memory_sqlite.recall` sin try/except alrededor de la llamada `memory_sqlite.recall(theme, k=4)` (solo el bloque completo) · Si `recall` lanza excepción en la segunda llamada, el script falla sin entregable, rompiendo el cron.

`parallel_loop.py` · ALTA · `_run_worker` llama a `llm_call.llm_call` sin verificar que `res` tenga atributo `model` antes de accederlo en `WorkerResult(model=res.model)` · Si `llm_call` retorna un objeto sin `model` (ej. error de red), causa `AttributeError` y el worker falla sin loggear correctamente.

`overnight_audit.py` · MEDIA · `_read_cluster` trunca archivos a `PER_FILE_CHARS` (6000) sin considerar que el límite real es `CLUSTER_CHARS` (20000) por cluster, no por archivo · Puede omitir código relevante de archivos grandes dentro del límite del cluster, generando auditorías incompletas.

`parallel_loop.py` · MEDIA · `recall_prior` captura `Exception` genérico en `memory_sqlite.recall`, pero no en `memory_sqlite` import (solo `except Exception: memory_sqlite = None`) · Si `memory_sqlite` falla después de importarse (ej. DB corrupta), `recall_prior` propaga la excepción y rompe la consolidación.

`nightly_digest.py` · BAJA · `read_runs` usa `if f'"ts":"{date}' in line` para filtrar líneas JSON · Si un campo `ts` contiene la fecha como substring (ej. `"ts":"2026-05-29T23:00:00Z"`), funciona, pero si hay otro campo con la misma fecha (ej. `"date":"2026-05-29"`), filtra falsos positivos · Ineficiente y propenso a errores; debería parsear JSON y verificar `ts.startswith(date)`.
